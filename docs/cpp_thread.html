编译环境与头文件

本模块介绍了如何在现代 C++ 项目中正确引入和链接线程库。脱离了直接调用 GCC 的命令行方式，使用 CMake 管理多线程项目是工业级开发的标准起手式。点击下方标签卡浏览不同的配置步骤。

核心 CMakeLists.txt 写法

寻找系统线程库的最专业、最稳妥的写法，替代写死的 -lpthread。

# 1. 声明 CMake 的最低版本要求
cmake_minimum_required(VERSION 3.10)

# 2. 设置项目名称
project(ThreadTest)

# 3. 寻找系统线程库 
find_package(Threads REQUIRED)

# 4. 指定生成的可执行文件名和源代码文件 
# ⚠️ 注意后缀一定是 .cpp，写成 .c 会报错！
add_executable(create_test create_test.cpp)

# 5. 将线程库链接到你的可执行文件
target_link_libraries(create_test PRIVATE Threads::Threads)

标准编译执行流程

在终端中（通常是在项目内新建的 build 文件夹里）执行以下指令：

# 第一步：生成配置 (检查环境并生成 Makefile)
cmake ..

# 第二步：正式编译 (调用编译器生成二进制文件)
make

# 第三步：运行程序
./create_test

🚀 创建即启动 / 带参数创建

招募工人，立刻执行

实例化 std::thread 对象时，线程会立即被操作系统调度并开始执行指定的任务函数。你可以按顺序将参数传递给任务函数。

// 无参数
std::thread t1(func_name);
// 带参数传递
std::thread t2(func, arg1, arg2);
⚠️ 传引用 (核心必考)

修改外部变量的必须手段

默认情况下，线程参数是按值拷贝传递的。如果你想让线程修改外部的变量（例如累加一个计数器），你必须使用 std::ref() 将变量包裹起来传递引用。

std::thread t3(func, std::ref(num));
🛑 等待结束 (阻塞)

绝对不能忘的 join()

主线程调用 join() 会被阻塞，死等子线程干完活再往下走。这能保证传给子线程的局部变量依然存活，直到子线程安全退出。负责回收系统资源，最常用！

t1.join();
崩溃警告： 在 C++ 中，std::thread 对象在销毁前必须明确其归宿（join 或 detach），否则程序会触发 std::terminate 导致直接崩溃。
👻 分离后台 (放养)

致命的诱惑：detach()

让线程自己在后台跑，变成守护线程，脱离主线程管控。分离后线程会在后台自动释放资源。

t1.detach();
避坑指南： 强烈不建议在安防等高稳定性项目中使用！如果子线程使用了主线程局部变量的引用，而主线程先结束销毁了该变量，子线程再去读写就会产生野指针/内存越界，导致极难排查的随机崩溃（Segment Fault）。
🛡️ 状态检查

防爆金牌：joinable()

一个线程不能被重复 join()。工业级代码通常会先使用 joinable() 进行安全检查，返回 bool 值表示线程是否还可以被归宿处理。

if (t1.joinable()) {
    t1.join();
}
⏱️ 跨平台休眠

告别系统的 sleep

让当前线程休眠指定的时间，实现完美的跨平台兼容性。

std::this_thread::sleep_for(std::chrono::milliseconds(10));

工业模板与数据竞争验证

本模块展示了在实际项目中如何批量管理线程，并提供了一个交互式图表模拟器，可视化证明多线程并发时不加锁（Mutex）带来的数据竞争风险。

用 Vector 批量管理线程

遇到需要开 10 个、100 个线程的场景，绝对不要用 C 语言的原生数组，直接套用这个 Vector 模板：

void worker(int& id) { 
    /* 干活逻辑 */ 
}

int main() {
    std::vector<std::thread> pool;
    int my_data = 100;

    // 1. 批量创建并派发任务
    for (int i = 0; i < 10; ++i) {
        pool.push_back(std::thread(worker, std::ref(my_data)));
    }

    // 2. 批量回收，等待所有工人下班
    for (auto& t : pool) {
        if (t.joinable()) { // 工业级安全检查
            t.join();
        }
    }
    return 0;
}
