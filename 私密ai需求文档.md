# 私密 AI 需求文档

## 1. 背景

当前项目已经有公开站点统一 AI，但它被明确限制为只能读取公开学习数据，不能触达私密留言板内容。

现在需要为 `secret_board.html` 单独增加一套“私密陪伴 AI”能力，用于承接我和男朋友在私密留言板里的生活化、情绪化、关系化对话。

这个 AI 不属于全站公共 AI，也不能复用公共 AI 的公开数据边界。

## 2. 目标

- 只在私密留言板内可见、可用
- 使用独立的服务端 API Key
- 能读取并理解私密留言板历史内容
- 能形成滚动记忆，而不是每次都只看单条消息
- 语气定位为温柔、克制、像朋友一样陪伴的情感类 AI
- 唤起方式不是按钮，而是在留言输入框中输入命令

## 3. 用户交互定稿

### 3.1 唤起方式

- 在私密留言板输入框中输入 `@小宝`，开启陪伴模式
- 输入 `@关闭`，关闭陪伴模式
- 开启后，后续普通留言发送成功后，AI 会自动跟进回复
- 如果输入的是 `@小宝 今天有点累` 这种形式，则视为：
  - 先开启陪伴模式
  - 再把后面的正文作为本次真实留言
  - AI 立即参与回复

### 3.2 显示方式

- AI 只在 `secret_board.html` 页面内出现
- 不增加全站悬浮按钮
- AI 回复直接作为留言流的一部分显示在私密留言板里
- AI 作者名固定显示为 `小宝`
- UI 需要与普通留言明显区分，但不能破坏现有页面风格

### 3.3 对话原则

- AI 默认不抢话、不连续刷屏
- 只有在陪伴模式开启时才参与
- `@关闭` 后，留言板恢复普通双人留言模式

## 4. 安全原则

### 4.1 API Key

- 私密 AI 的 API Key 不允许写入前端源码
- 私密 AI 的 API Key 不允许写入普通 Supabase 表
- 必须放在 Supabase Edge Function 环境变量中

### 4.2 私密数据访问

- 私密留言板内容不能再依赖前端直接使用 anon key 读写
- 私密留言板的读取、发送、AI 调用都改为走服务端函数代理
- 私密留言板访问口令不再硬编码在前端页面里

### 4.3 现状风险与本轮处理

当前项目的“管理员身份”和“隐藏身份”都是前端态，不足以保护真实密钥和私密数据。

因此本轮私密 AI 落地时，顺带把私密留言板访问改成：

- 前端输入私密口令
- 服务端验证口令
- 服务端签发短时访问 token
- 后续留言板读取、发言、AI 对话都带这个 token

这样虽然仍然是共享口令模式，但至少：

- 私密口令不再裸露在前端源码里
- 私密 AI key 不会暴露到浏览器
- 私密留言数据可以从公开 anon 访问中移开

## 5. 服务端设计

新增 Supabase Edge Function：`secret-companion`

### 5.1 action 设计

- `status`
  - 返回私密留言板服务和私密 AI 是否配置完成
- `auth`
  - 校验私密口令
  - 返回短时访问 token
- `load-messages`
  - 根据访问 token 读取私密留言板历史
- `post-message`
  - 根据访问 token 写入一条人工留言
- `chat`
  - 根据访问 token 读取私密留言历史与记忆摘要
  - 调用私密 AI
  - 把 AI 回复写回 `secret_messages`

### 5.2 环境变量

- `SERVICE_ROLE_KEY`
- `SECRET_BOARD_PASSWORD`
- `SECRET_BOARD_TOKEN_SECRET`
- `SECRET_COMPANION_API_KEY`
- `SECRET_COMPANION_API_ENDPOINT`
- `SECRET_COMPANION_MODEL`

## 6. 数据结构

### 6.1 现有表复用

继续使用 `secret_messages`

留言内容仍然保存为 JSON 字符串，但会补充 `type` 字段区分：

- `human`
- `private_ai`

### 6.2 新增记忆表

新增表：`secret_ai_memories`

用途：

- 保存私密关系上下文的滚动摘要
- 记录已经处理到哪一条留言
- 避免每次都把全部历史直接塞给模型

建议字段：

- `scope`
- `memory_summary`
- `last_message_id`
- `updated_at`

## 7. 记忆策略

### 7.1 短期上下文

- 每次 AI 回复时读取最近一批私密留言
- 重点保留最近对话语气、事件、情绪变化

### 7.2 长期记忆

- 服务端维护一份滚动摘要
- 当出现新的人工留言后，按增量更新摘要
- 摘要内容聚焦：
  - 两人的关系语境
  - 最近生活事件
  - 情绪状态变化
  - 反复提到的重要人、地点、计划、烦恼、惊喜

### 7.3 记忆边界

- 私密 AI 只记住私密留言板自己的内容
- 不读取全站公共 AI 历史
- 不读取爸妈留言
- 不读取管理员后台内容
